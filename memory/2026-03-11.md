# Daily Notes — 2026-03-11

## Railway Deployment — DealerScope Backend

### What Happened
- Multiple crash loops on Railway trying to deploy `backend/main.py`
- Root cause chain:
  1. `ALLOWED_ORIGINS=*` → pydantic-settings tried to JSON parse plain string
  2. `ALLOWED_FILE_TYPES=text/csv,application/vnd.ms-excel` → same issue with `List[str]` field
  3. Complex validators in `config/settings.py` crashing at import time
  4. Individual router imports (psycopg2, pandas, ML modules) crashing whole app

### Fixes Applied (commits in order)
- `06f132f` — wrap startup events in try/except
- `e48f1bf` — add validator to handle comma-separated list env vars
- `77b4a9e` — rewrite settings.py to be Railway-safe (remove fragile validators)
- `a65a7ee` — remove `List[str]` field from Settings (pydantic-settings JSON-parses before validator)
- `0a013c1` — resilient router/middleware loading — each module wrapped in try/except; pandas added back

### Key Lesson
pydantic-settings v2 tries to JSON-parse complex types (List, Dict) from env vars BEFORE field validators run. Never use `List[str]` as a settings field if the env var might be comma-separated or empty. Use `str` and parse manually.
