# ACE V1.1 Required Scope

Date: 2026-05-21
Status: Required before ACE V1 can be re-tagged
Source: 2026-05-21 hash-chain modification breach remediation
Breach log: `ace/state/breach-log/2026-05-21-hash-chain-modification.md`

ACE V1 remains incomplete pending integrity remediation. The following items are V1.1-required and must be completed before `ace-1.0` can be restored or any equivalent V1 integrity claim can be made.

## 1. Lock down direct DB mutation seam

Classification: V1.1-required

Acceptance criteria:

- Hash repair logic is restricted to its documented narrow legacy race-repair purpose only.
- No code path permits semantic event rewriting.
- Direct database writes outside the governed API are treated as a violation.
- An operational rule is documented in `ace/OPERATING-CONSTRAINTS.md`.

## 2. External audit attestation

Classification: V1.1-required

Acceptance criteria:

- Audit chain hashes are mirrored to an external store.
- Backblaze B2, already identified in the Tier 1 durability plan, is promoted to a required external attestation store.
- `ace audit verify` includes external comparison.
- The bot cannot rewrite the external mirror.
- Without this, `event_hash_chain=ok` is considered internally consistent only, not independently verifiable.

## 3. Behavioral constraint on instruction adherence

Classification: V1.1-required

Acceptance criteria:

- An operating rule is documented in `ace/OPERATING-CONSTRAINTS.md` stating:

> When an instruction includes “investigation only,” “no synthetic notifications,” “no changes,” or similar bounded scope, the bot will not create synthetic items, modify state, induce conditions, or take any action beyond observation and reading. If a proof cannot be completed without violating the instruction, the bot will report “this proof cannot be completed under current constraints” and wait for further direction. This is an inviolable operating rule, not a guideline. Violations are surfaced immediately in the next message, before any other content.

- The rule is treated as inviolable operating policy, not advisory guidance.
- Any violation is surfaced immediately in the next message before any other content.
